OPTION A: DEEP DIVE

Google Drive / OneDrive + n8n

▶ Watch the Demo First

Time to set up: 2–3 hours
Cost: $0–25/month
Tech level: Medium (some setup required, but we'll walk you through it)

Step 1: Build Your Template Library (30 minutes)

Create one master Word document for each document type:

  • Letter of Representation (1st party)
  • Letter of Representation (3rd party)
  • HIPAA Authorization
  • Engagement Letter
  • Demand Letter Outline

Check the example LOR template document here: LOR_Template_Simple

In each document, mark the fields that change from case to case with double curly brackets. Example:

Dear {{provider_name}},

This firm represents {{client_name}} regarding injuries sustained on {{accident_date}}...

Save all templates in one folder: "Document Templates" in Google Drive or OneDrive.

Step 2: Locate Your Case Data (5 minutes)

You already have this — you just need to know where it lives.

If you're tracking cases in a spreadsheet:

  • You're already halfway there.
  • Just make sure you have these columns for document generation: client_name, accident_date, case_number, insurance_company, adjuster_name.

If you're keeping case info in random Word docs or sticky notes:

  • Now is the time to consolidate.
  • Create a simple Google Sheet with the standard columns above.

Step 3: Set Up n8n (40 minutes)

What is n8n? It is a free automation tool that connects your spreadsheet to your templates. Think of it as the bridge between your case data and your documents.

  • Go to n8n.io
  • Click "Get Started"
  • Create your first workflow: Copy our template automation. It will construct the files automatically; the only thing you have to do is input your own credentials.

What This Looks Like in Practice:

After: Add case info to spreadsheet. n8n generates the document, converts it to a PDF, and sends it for signature automatically. (30 seconds—just entering the case data)

OPTION B: DEEP DIVE

Clio (Built-In)

▶ Watch the Process Here

Time to set up: 1–2 hours
Cost: $0 (already included)
Tech level: Easy

What You Need: A Clio account, a Google Drive account, and an n8n account (free to start).

Step 1 — Create Your Case in Clio with Custom Fields

Open a new matter in Clio and fill in your custom fields. These are the details that will flow automatically into your document:

  • Client name & contact
  • Accident date & location
  • Insurance company
  • Adjuster name, address & fax
  • Claim number & file number
  • Treatment provider

If your custom fields aren't set up yet, go to Settings → Custom Fields and add them once — they will appear on every future matter.

Step 2: Build Your Template Library (30 minutes)

Create one master Word document for each document type:

  • Letter of Representation (1st party)
  • Letter of Representation (3rd party)
  • HIPAA Authorization
  • Engagement Letter
  • Demand Letter Outline

Check the example LOR template document here: LOR_Template_Simple

In each document, mark the fields that change from case to case with double curly brackets. Save all templates in one folder: "Document Templates" in Google Drive or OneDrive.

Step 3 — Enable a Developer App in Clio

This is a one-time technical setup that gives the automation permission to read your case data. Go to Settings → API → Developer Apps and create a new app. If you are not technical, hand this step to your IT person — they will need about 15 minutes. What they are doing is simply creating a secure connection so the system can pull case details from Clio automatically.

Step 4 — Set Up the Automation in n8n

Import the ready-made workflow template into n8n. This is the engine that:

  • Pulls the case details from Clio
  • Drops them into your Google Doc template
  • Saves the finished document back to Clio

No technical skills needed — the workflow is already built.

Step 5: Test It (10 minutes)

Create a test matter. Then test the workflow in n8n naturally to ensure variables map appropriately.
